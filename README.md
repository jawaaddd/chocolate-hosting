# Chocolate Hosting

A Minecraft vanilla server hosting platform to make playing with friends accessible, fast, and trivial!

There's a problem with the all Minecraft server hosting platforms out there... For the casual player who wants to play survival with some friends, they need to figure out how much RAM they need, how many CPU cores (does it even matter?!), and how to fiddle with the server properties to get the experience they want.

And they still find themselves falling into a pit of lava and losing their hard earned diamonds because of lag.



For a lot of players, it doesn't matter whether they have the latest and greatest plugins for teleporting back to their base, the craziest modpacks which turn their world into Game of Thrones, or if they have 8GB of RAM. What matters is that they can call their friends on Discord, ask if they want to build a house together, and be able to do so easily and quickly, without worrying about lag and performance.



That's the goal. Giving players a platform where they can start setup a server within 5 minutes and 5 clicks, without sacrificing performance and a smooth experience.



The target audience for Chocolate Hosting is the simple and casual player. Not the sophisticated one who wants all the features I mention above. That's why this platform is only offering vanilla servers. Well, almost vanilla (I have some ideas on using Fabric for optimizing my servers and making them more cost-efficient), which is why this project is called *chocolate* hosting and not vanilla hosting.



## Development

This project is still in development, but hopefully I remove this line by the time you next read this README. To follow along with this project's development, take a look at the ```DevelopmentPlan.md``` file in the ```backend``` folder.

This is a showcase of my skills as a programmer. So here's the tech stack:

* **Golang** for the web server and server manager
* **PostgreSQL** for the database
* **React + ReactNative** for the website and mobile apps
* **AWS** for the hosting infrastructure



I also decided to avoid using AI to write any code for me, although I'm definitely using it for learning. I decided this because I wanted this project to be a showcase of my programming fundamentals and for a peek into how my brain works as I architect a large project like this.

I have other projects on my GitHub where I do heavily use AI as the focus for those is rapid-development, experimenting, and prototyping.
